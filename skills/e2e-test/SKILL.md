# E2Eテスト

## 概要

Playwright MCPでローカル開発サーバーに接続し、画面操作を伴う動作確認を行う。
主に `docs/web/*.html` の回帰確認、UI崩れ確認、入力反映確認を対象とする。

## 接続方法（localhost:8080）

1. 事前にローカルサーバーを起動する（例: `http://localhost:8080/`）。
2. Playwright MCPで `http://localhost:8080/` に `navigate` する。
3. 対象ページ（例: `http://localhost:8080/docs/web/skill_inherit.html`）へ遷移する。
4. `snapshot` で要素参照を取得してから操作する。

## 手順

1. 対象ページを開き、初期表示の主要結果を取得する。
2. 入力、チェック切替、ボタン押下、タップ相当操作を行う。
3. 操作後の結果テキスト・入力値・要素状態を取得し、初期状態との差分を確認する。
4. 必要に応じてスクリーンショットを取得し、見た目の崩れを確認する。
5. コンソールエラーを確認し、機能影響のあるエラーを優先して報告する。

## チェック観点

- 入力変更で結果がリアルタイムに更新されるか。
- 追加・削除操作で行数や結果表示が追従するか。
- アコーディオン、カルーセルなどの開閉・遷移操作が機能するか。
- モバイル相当サイズで表示崩れがないか。
- URLパラメータ受け取りが初期表示へ反映されるか。

## 注意点

- `file://` は使わず、必ず `http://localhost:8080/` 経由で確認する。
- 接続失敗時は、サーバー起動状態とURLを先に確認する。
- ブラウザキャッシュの影響が疑われる場合はハードリロードで再確認する。
- 404の `favicon.ico` など、機能に無関係なノイズは分離して報告する。
